The Form Element acts as a container for all the fields and serves to associate them together as a single group of inputs.

The Action Attribute holds the file path to a script that receives and handles the data sent from the form.

The Input Element is a self-closing tag and uses attributes to determine its behavior.

The Type Attribute tells the input element what kind of input field it should be. Some examples include:
























The Textarea Element creates a large multi-line text field. It requires opening and closing tag, and any text between the two tags will auto-fill the field.

The select element creates a drop-down field and the option element.
When nested inside a select, provides the selection options. Both elements require opening and closing tags.


The Value Attribute shows the user something different for an option than the data that gets recorded when the form is submitted

To show a grayed-out first option in select like "Choose one", you can use the attributes
disabled to prevent it from being selected, the hidden to prevent it from showing in the dropdown,
and the selected to make sure it shows by default.


The Name Attribute identifies the data of each field in a form.
To provide this information, it is important to include a name attribute that identifies the data of each field in a form.

The name attribute is similar to the label element discussed next in that it identifies the different form fields.
But while the label element identifies a field for human users, the name attribute identifies the field for the computer that processes the submitted form.

The Label Element is used ot identify each field, and should include the for attribute which should match the id
applied to its form field. This lets the browser know which label is for which field.


Submit buttons
The input element can be used by specifying the corresponding type attribute.
In this case, type="submit". By default, the button text reads "submit", but that text can be changed using the value attribute.

The button element can be used to submit a form, but can also be used to trigger javascript actions.
To use it for a form, use type="submit" as an attribute. One of the main differences between the button element and input is that
button needs a separate closing tag. Whatever text is placed between the opening and closing tags is the text that is shown on the button.

You can also use the Input Element for a button.

The required attribute can be added to any form field.
If that field is left empty when the user attempts to submit, the browser will show an error saying the field must be filled.

The disabled attribute can be added to any form field.
It effectively "turns off" the functionality of that field and does not allow the user to interact with it.

The placeholder attribute accepts text or another appropriate value to show as a "placeholder" to indicate where and how the user should add input.
